# /diagnose — 診断器

## コア思想

> KOSは「他人の物語を解説する道具」ではなく「自分の現在地を測る道具」になり得る。

232パターンを**自分の状況に向ける**。読み物として消費するのではなく、判定結果として使う。出口は「打てる手」。

## 使い方

```
/diagnose <状況の記述（3〜5行）>
```

例:
- `/diagnose 100日プロジェクトをDay 62まで続けたが、スキ数が伸びない。読者層は明確だが、書く方向性に迷いが出てきた。`
- `/diagnose チームで自分が責任者だが、上から下りてくる方針と現場の判断が食い違う。発言すれば角が立ち、黙れば現場が崩れる。`

---

## フロー

### Phase 1: 構造抽出

入力テキストから以下を引き出す（足りなければ質問する）:

| 軸 | 質問 |
|----|------|
| 主体 | 誰が誰に何をしている／されている |
| 力の方向 | 上下／対等／逆転途中 |
| 時間軸 | 始まった時期と現在のフェーズ |
| 欠落 | 構造的に**ない**もの（権威の根拠／撤退ライン／フィードバック）|
| 痛み | どこに違和感があるか（症状）|

不足が多い場合は最大3問だけ追加質問。**質問しすぎない**。

### Phase 2: パターン照合（top 3）

`dictionary/patterns/` 全件と grep / 構造照合:

- 各候補に **信頼度（high/mid/low）** と **根拠** を付ける
- 競合候補が出てくるのは健全。1つに絞らず3つ並べる

### Phase 3: Playbook引当

`(optional) your own playbook registry` で照合:
- マッチするPlaybookがあれば → そのDecision Algorithmを当てる
- なければ → 「該当なし。新Playbook候補」とマーク

### Phase 4: 判定

下記4種から判定種別を選ぶ:

| 判定 | 内容 |
|------|------|
| GO | 続行。次の閾値まで様子見 |
| HOLD | 一時停止。観測指標を増やしてから再判定 |
| PIVOT | 方向転換。同じ目的を別の構造で達成 |
| EXIT | 撤退。これ以上のコストは沈む |

### Phase 5: 打てる手

時間軸別に最大3つずつ:
- **即時（24時間以内）**
- **1週間以内**
- **撤退ライン（これが起きたらEXIT）**

実行可能な行動だけ。「考える」「検討する」は書かない。

## 制約・注意

1. **判定を盛らない**。GO は「証拠あり」、HOLD は「データ不足」が正直な判定であることが多い。
2. **打てる手は実行可能な動詞で**。「方向性を考える」「整理する」はNG。「Xに会って話す」「Yをドロップする」がOK。
3. **撤退ラインを書かない診断は無効**。撤退条件が決まらない判定は判定ではない。
4. **同情モードを禁止**。「大変ですね」は要らない。構造として何が起きているかだけを書く。
