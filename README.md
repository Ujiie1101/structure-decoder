# structure-decoder

テキストを構造で読むための、辞書 × エンジン。

書物・記事・目の前の状況を、蓄積された **atoms（構造原子）** と **patterns（構造パターン）** を通して分解・命名する。個人の認識フィルターを、誰でも引ける形に変換する公開プロダクト。

> 「材料」ではなく「料理とレシピ」を出す。
> 一次資料は持ち込まず、抽出された構造辞書と、それを動かす分解エンジンだけを公開する。

## なぜこれを作るか

AIが要約・翻訳を担う時代に残された人間の仕事は、**「何をやるべきかを自分で見つけ、構造として名指す」** こと。

generic な LLM は「この文章を要約して」には答えるが、「この文章にどの構造パターンが起動しているか」には答えない。なぜなら、その**事前辞書**を持っていないから。

structure-decoder は、数百冊の読書と数百日の観察から抽出された辞書を、誰でも引けるようにする。

## 構成（予定）

```
structure-decoder/
├── dictionary/        # = 脳そのもの（atoms / patterns / mechanisms / meta）
├── engines/           # = 分解エンジン（Claude Code skills として実装）
├── examples/          # = 装置が動いた証拠（書物の章別分解サンプル）
└── docs/              # = 方法論とアーキテクチャ
```
