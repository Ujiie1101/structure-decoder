# 異論抑圧

集団内で異論を表明した者に対し、他のメンバーが社会的罰（嘲笑・無視・排除）を加える外圧。横軸の罰メカニズム。

## 起源（Why this exists）

集団の凝集性は、共通信念を共有することで担保される。共通信念から逸脱する者は、集団凝集性への脅威として認識される。メンバーは（無意識に）凝集性を守るため、逸脱者に対する罰を執行する。

この罰は中央集権的に課されるのではなく、メンバー間で**分散的に**発動する。具体的には、目を合わせない、嘲笑する、議題から外す、私的な評判を下げる、といった微細だが連続的な制裁。

罰の存在自体が、観察している他のメンバーに「異論を出さない」シグナルを送る。これにより異論抑圧は集団の自己強化的なルールとなる。

## 観察可能なproxy

- 異論者の発言後の集団の沈黙時間
- 異論者の昇進・配属パターン
- 会議外での異論者への評価言説
- 退職率と発言頻度の相関

## 介入点（Disable conditions）

- リーダーによる異論奨励の明示化（異論を出した者を公的に評価）
- 異論を制度化（red team、pre-mortem）
- 集団境界の透過化（外部メンバー、ローテーション）
- 罰の検出と懲戒（異論者への報復行為のメタ罰）

## 関与パターン

- [groupthink](../patterns/groupthink.md) — 外圧として作動する
- （拡張候補: holy-war, herd-behavior, retaliatory-spiral）

## 補足

[conformity-pressure] と [dissent-suppression] のセットは、**自己検閲**を完成させる:

```
1. 異論を持つ
2. 過去に異論者が罰された記憶を想起 → dissent-suppression の予期
3. 自分も同じ罰を恐れる → conformity-pressure に屈する
4. 黙る
5. 他のメンバーから見ると全員一致に見える → illusion-of-unanimity
```

このループがgroupthinkの完成形。介入は1〜4のどこかで止める必要がある。
