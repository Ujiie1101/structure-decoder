# ダンバー限界

人間が**安定した社会関係**を維持できる人数の認知的限界。約 **150 人**（Dunbar 1992）。この閾値を超えた集団は、対面関係ベースでは機能せず、制度・階層・抽象的ルールに依存せざるを得なくなる。

## 起源（Why this exists）

Robin Dunbar（1992）は霊長類の **新皮質サイズ** と **集団規模** の相関から、人間の社会関係の認知容量を予測した。約150人は、**狩猟採集集団の典型的サイズ**、**ローマ軍の中隊規模**、**現代の機能する企業の上限規模**などと一致する。

ダンバー数は単一の数値ではなく、**階層構造**を持つ:
- 5人：親密な関係
- 15人：支援を得られる関係
- 50人：頻繁に会う関係
- 150人：安定した関係を維持できる上限
- 500人：知り合い
- 1500人：顔と名前が一致する上限

これを超えた集団では、**個人レベルの相互認識**が崩れ、**役割・階級・職位**による匿名的相互作用にシフトする。

## 観察可能なproxy

- 組織規模と意思決定速度の関係（150人を超えると官僚化）
- SNS のフォロワー数と実質的関係数の乖離
- 「あの人、誰だっけ」現象の頻度
- 大企業の階層数（150人ごとに新階層が必要）
- 軍隊の編成（中隊 = 約150人）

## 介入点（Disable conditions）

- 集団規模の意図的縮小（150人以下にユニット化）
- ダンバー階層を意識した組織設計（5/15/50/150 で役割分離）
- ※ ダンバー限界そのものは認知容量なので、超えることは不可能

## 補足

[scale-mismatch]（既存、hubris-nemesis）の**進化的根拠**。なぜ scale-mismatch が起きるか？人間の認知能力は約150人規模に最適化されており、それを超えた規模では**質的に異なる処理様式**が必要になる。

[complexity-overwhelm] とも関係。複雑性の多くは関係性の多重から発生し、関係性の処理は dunbar-limit に縛られる。

algorithmic-abdication が「規模が認知を超える」ことを核とする以上、dunbar-limit はその**生物学的根拠**として位置付けられる。スケール超過を解消する方法は、AI 委譲（abdication）か、規模縮小（モジュール化）の二択しかない。
