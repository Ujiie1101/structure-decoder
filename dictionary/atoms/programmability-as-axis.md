# プログラマブル軸

道具やサービスを評価するとき、「コードから呼び出せるか／プログラマブルに駆動できるか」という軸。量産業務・自動化業務を持つ事業者には決定的に効くが、単発業務しか持たない事業者には可視化されない、文脈依存的な評価軸。

## 起源（Why this exists）

道具は本来、人間が手で扱うものだった。産業革命以降、機械化された道具が登場し、「人が道具を直接操作する」と「人が機械を介して道具を操作する」が分岐した。情報技術以降、その分岐がさらに進み、「人が道具を呼び出す」と「コードが道具を呼び出す」が並走する。

プログラマブルかどうかは、道具の物理的・技術的特性だが、評価軸として機能するかどうかは業務側の文脈に依存する。量産が必要な業務にとっては最重要軸、単発業務にとっては無関係な軸。

## 観察可能なproxy

- API・CLI・SDK の有無
- データ駆動の入力（CSV、JSON、DB クエリ）受け入れ可否
- CI/CD への統合可否
- スクリプトからの呼び出し例の有無
- 「自動化事例」の流通量

## 介入点（Disable conditions）

このatomは「軸」なので「打ち消す」対象ではない。代わりに：

- **業務文脈の特定**: 量産業務を持つかどうかを言語化する
- **量産業務の数値化**: 「週次○件」「月次○件」と数値化する
- **API化要求の表明**: 候補ツールに API 提供を要求する／無ければ候補外にする

## 補足
