# 状態結合

あるノードの状態変化が、接続先ノードの状態を強制的に変える関係。情報伝達ではなく、状態の連動。

## 起源（Why this exists）

物理系では力学的結合として常在（連結された振り子、結合された電気回路）。社会・経済系では、契約・与信・信頼関係が状態結合を作る。Aが破綻すればBの債権が消える、Aが裏切ればBの信用が損なわれる、という形で「状態が状態を引きずる」関係が生まれる。

情報伝達と異なり、状態結合は受け手の解釈を介さない。Aの破綻はBの貸借対照表に**自動的に**穴を開ける。これが連鎖の機械性を生む。

## 観察可能なproxy

- 二者間の与信・債権・依存関係の存在
- 「Aが倒れたらBも倒れる」という条件付き予測の妥当性
- 株価相関係数、CDSスプレッド連動性
- 共通設備・共通サプライヤーへの依存

## 介入点（Disable conditions）

- 結合の弱化（保証・保険による緩衝）
- 結合の透明化（誰が誰に依存しているかの可視化）
- 同期解除（取引タイミングのランダム化、ロックアップ期間）
- 結合切断（取引停止、隔離措置）

## 関与パターン

- [cascade](../patterns/cascade.md) — 連鎖の機械的駆動力
- （拡張候補: contagion, network-parasitism, exit-blockade）

## 補足

接続密度（[interconnection-density]）と混同されやすいが別物。接続があっても結合が弱ければ連鎖しない（インターネット上の弱い情報接続）。結合の強さは接続経路の物理性・契約性・自動性に依存する。

WWIの同盟網は「条約義務」という強い state-coupling を持っていた。同じ接続密度でも、文化的同盟（緩い結合）なら連鎖は限定的だっただろう。
