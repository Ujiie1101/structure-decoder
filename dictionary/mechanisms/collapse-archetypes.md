# Collapse Archetypes v0.1

## Collapse Spectrum Model

```
            SPEED
            ↑
    fast    │   ┌─────────────────┐
            │   │ collapse-cascade │
            │   │  • high connectivity
            │   │  • rapid propagation
            │   │  • low reversibility
            │   └─────────────────┘
            │
            │         threshold
            │         proximity
            │            ↓
            │   ┌─────────────────┐
    slow    │   │ collapse-erosion │
            │   │  • low connectivity
            │   │  • gradual decay
            │   │  • moderate reversibility
            │   └─────────────────┘
            └────────────────────────→ TIME
                 days    years    centuries
```

## Dimensional Analysis

| Dimension | collapse-cascade | collapse-erosion |
|-----------|------------------|------------------|
| **Speed** | Fast (days–weeks) | Slow (years–centuries) |
| **Connectivity** | High (dense network) | Low (peripheral actors) |
| **Reversibility** | Low (cascade locks in) | Moderate (reform possible) |
| **Modularity** | Low (tightly coupled) | High (loosely coupled) |
| **Trigger** | Threshold breach | Symbolic recognition |
| **Propagation** | Network contagion | Function migration |

## Universal Sequence

Both archetypes follow the same abstract sequence:

```
erosion/accumulation → threshold approach → breach/recognition → cascade/fragmentation
```

The difference lies in **timescale** and **connectivity density**.

## Usage

- Use `collapse-cascade` when: rapid propagation, network effects, trust/liquidity shock
- Use `collapse-erosion` when: prolonged decay, legitimacy loss, function transfer to periphery
- Use generic `collapse` when: classification unclear or both dynamics present
