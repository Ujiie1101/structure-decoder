# design（設計）

意図的な計画と構想により、目的を達成するための形態・構造・システムを創出する構造。

## 定義

問題の理解、制約条件の把握、創造的解決策の考案を通じて、機能・美・実現可能性を統合した人工物やシステムを計画的に生み出す。

## 構造

```
problem_understanding
  → constraint_analysis
    → creative_synthesis
      → prototype_iteration
        → implemented_solution (form follows function)
```

## 発動条件

1. 明確な目的・要件の定義
2. 制約条件の認識
3. 創造的問題解決能力
4. 反復的改善プロセス

## 失敗条件

- **要件不明**: 問題定義の不備
- **過剰設計**: 不要な複雑性の導入
- **美的偏重**: 機能性の軽視
- **硬直化**: 使用段階のフィードバック無視

## 適用事例

| Layer | Entities |
|-------|----------|
| technology | user-interface, software-architecture, industrial-design |
| art | bauhaus, design-thinking |
| economics | mechanism-design, market-design |
| philosophy | teleology, intentionality |

## 関連パターン

- **emergence**: 設計から予期せぬ特性が創発
- **constraint**: 制約が設計を形成
- **feedback**: 設計改善の循環
