# enabling（可能化）

新しい条件・技術・制度の出現が、以前は不可能だった行動・創造・組織を可能にする構造。

## 定義

インフラ、技術、制度、知識などの基盤的条件が整うことで、それまで存在し得なかった活動・組織・創造物の出現が可能になる。可能化は直接的な因果ではなく、潜在性の解放である。

## 構造

```
foundational_condition_emergence
  → barrier_removal
    → latent_potential_release
      → new_activity_proliferation
        → emergent_ecosystem (unplanned by enabler)
```

## 発動条件

1. 制約条件の緩和・除去
2. 新しい基盤技術・制度の普及
3. 潜在的需要・欲求の存在
4. 実験・試行の許容

## 失敗条件

- **過剰規制**: 新しい可能性の抑圧
- **独占**: 基盤へのアクセス制限
- **無関心**: 潜在需要の不在
- **インフラ未整備**: 基盤条件の不足

## 適用事例

| Layer | Entities |
|-------|----------|
| technology | internet, smartphone, api-economy |
| economics | limited-liability, stock-exchange |
| art | printing-press, recording-technology |
| history | railroad, electricity-grid |

## 関連パターン

- **emergence**: 可能化から新秩序が創発
- **platform-adjacency**: プラットフォームによる可能化
- **scaling**: 可能化がスケールを解放
