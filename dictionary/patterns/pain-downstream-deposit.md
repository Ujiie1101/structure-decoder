# Pain Downstream Deposit（痛みの下流堆積）

## 定義

組織を流れる責任とコストは、上流から下流に重力で落ちる。痛みは最下流に溜まる一方、それを言語化する力は最上流に偏る。**認識の勾配と痛みの勾配は、逆向きに走る。**

externalization-cascadeが「コストの流れ方」を記述するのに対し、本パターンは**「認識と痛みの非対称」**そのものを焦点化する。

## 公式

```
組織の階層構造
   ↓ 業務委託・再委託・最適化
痛みは重力で下流に落ちる
   ↑ 一方
言語化の力は重力に逆らって上流に偏る
   ↓ 結果
最下流: 痛い + 言語化できない
最上流: 痛くない + 言語化できる
   ↓
組織の認識: 「問題は起きていない」
現場の体感: 「何かおかしい」
```

## メカニズム

### 1. 監視の階段状の薄化

```
[本体] ── 監視チーム + ダッシュボード（厚い）
   ↓
[ベンダー] ── 監視（中)
   ↓
[ベンダーの下請け] ── 監視（薄い）
   ↓
[末端] ── 監視（ほぼ無い）
```

最適化フェーズで真っ先に削られるのが、最下流の監視である。「重要度が低い」と判断されるが、実際は**「重要度が低い」ではなく「上流から見えにくい」**だけ。

### 2. 攻撃と劣化が一番薄いところから現れる

外部攻撃も品質劣化も、システムの最も薄い場所から発現する。これは末端の能力が低いからではなく、構造的に**監視の手厚さと痛みの発生確率が反比例**するからである。

### 3. 平均値メトリクスと個別体感の乖離

上流は「平均値」で品質を測る。
最下流のユーザー/作業者は、自分の使い方の体感で判断する。
平均では検知されない異常が、**個別体感では先に顕在化**する。
だから「ユーザーが先に気づく」現象が常に起きる。

### 4. 言語化能力の階段状の偏在

| 階層 | データ | 会計 | メディア接続 | 言語化能力 |
|---|---|---|---|---|
| 上流 | あり | あり | あり | 高 |
| 中流 | 部分的 | 部分的 | 部分的 | 中 |
| 下流 | なし | なし | なし | 低（違和感はあるが構造化できない） |

末端は「違和感」までは持つが、それを構造として言語化する手段を持たない。

## 発動条件

1. 階層構造が3層以上（本体→委託→末端、など）
2. 各層が「自分の領域の最適解」を独立に追求している
3. 監視・計測の手厚さが上流に偏っている
4. 末端からのフィードバック経路が形式化または不在

## 失敗条件（堆積が解消される条件）

- 末端のフィードバックが上流に直結するチャネル（ポストモーテム、苦情公開、現場ラウンド）
- 計測指標に「分布の端」を含める（平均だけでなくp95, p99, 個別ケース）
- 上流が定期的に末端の現場に降りる文化
- 自分転倒の文化（末端からの違和感を上流が公的に拾う）

## 判別法

1. 異常を最初に報告するのが**いつも末端ユーザー/末端作業者**である
2. 上流のダッシュボードでは「正常」が示されている
3. 末端は「何かおかしい」と感じているが、原因を特定できない
4. 問題が認識される頃には、既に外部に漏れている

4つ揃うと痛みの下流堆積が作動している。

## 事例

| Layer | Entities |
|-------|----------|
| AI industry | Anthropic Claude Code品質低下（ユーザー報告が先行） |
| AI industry | Anthropic Mythos侵害疑義（third-party vendor環境経由） |
| supply chain | サプライチェーン下流の品質事故（自動車リコール等） |
| labor | 末端労働者の労災・メンタル不調 |
| customer service | コールセンター現場の顧客クレーム集積 |
| public health | 医療現場の医療過誤の構造的隠蔽 |

## 関連パターン

- **externalization-cascade**: コストの流れ方を記述（本パターンの上流）
- **fragility**: 最下流が最も脆弱な構造
- **information-asymmetry**: 情報の偏在の一形態
- **self-undermining-success**: 成功による階層化が下流堆積を加速
- **self-tipping**: 下流堆積を上流が能動的に拾い上げる戦略
- **parental-deposit**: 痛み（しつけ・摩擦）を親が下流（学校・国家）に堆積させる家族版（Day62）

## 核心洞察

> **本社は鈍感で末端は敏感。痛みは下流に堆積し、言語化の力は上流に偏る。**
> **異常を最初に報告するのが末端であるシステムは、本社のダッシュボードが「正常」を示し続けている間、構造的に治癒しない。**
