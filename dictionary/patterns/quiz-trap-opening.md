# Quiz-Trap Opening Pattern（3択罠導入パターン）

## 定義
3つの選択肢（A/B/C）を提示し、「常識的にAかBだろう」と読者に同意させてから、実際の答えがCであることを明かす導入手法。

## 公式
```
「Xが起きたら何が起きるでしょうか？」
A. [常識的期待]
B. [静かな結末]
C. [衝撃的現実]
↓
「多くの人はAかBを選ぶでしょう。当然です。」（同意を取る）
↓
「しかし実際に起きたのはCです。」（裏切る）
```

## メカニズム

### 1. 前傾姿勢の構築
読者を「答えを選ぶ」プレイヤーにすることで、受動的な閲覧から能動的な参加に切り替える。

### 2. 同意→裏切りの認知コスト
一度「当然です」で同意させることで、Cの衝撃度が上がる。
→ 予測が外れた分、残りの本文を「なぜCなのか」を知るために読む動機が生まれる。

### 3. 三者択一の限定
AとBが常識的選択として機能し、Cが「例外」として際立つ。
2択では対立構造になるが、3択は「常識 vs 例外」構造を作れる。

## 成功条件
- Cが**実在する事件**である（捏造は不可）
- AまたはBが**本当に多数派の予測**である
- クイズが本文テーマと直結している

## 失敗パターン
- Cが恣意的すぎる → 読者が「後出しジャンケン」と感じる
- AとBが弱すぎる → Cの衝撃が薄れる
- クイズと本題が遊離 → 導入として機能しない

## 事例

### Day 25「99万人の沈黙」
```
「世界平和を願うイラストを投稿したら？」
A. 共感が集まる
B. 静かにスルーされる
C. 「反日」と攻撃され謝罪に追い込まれる
```

## 適用条件
- 常識が通用しない異常事態を扱う記事
- 構造的問題を可視化したい記事
- 3000字級の長文記事（短文だと導入が重くなる）

## 注意点
- 頻繁に使うとパターン化して効果が薄れる
- Cは「ありえない」ではなく「ありえるが信じたくない」レベルに設定する
