# recall-anchor（想起アンカー）

日常物に意味を埋め込み、読者の生活動線で想起を誘発する設計パターン。

## 定義

記事やコンテンツの核心を、読者が毎日視界に入れる日常物に結びつける。読者は「わかりやすい比喩」として受け取るが、実際にはその物を見るたびにコンテンツを想起する装置として機能する。

## 構造

```
content_core (伝えたい構造・洞察)
  → daily_object_selection (毎日視界に入る物を選ぶ)
    → natural_connection (比喩として自然に結びつける)
      → embedding (記事内に「例え」として登場)
        → recall_trigger (読者が日常でその物を見る)
          → content_recall (記事が浮かぶ)
```

## 発動条件

1. **高頻度接触**: 選んだ物が読者の日常で頻繁に視界に入る
2. **自然な接続**: 核心と日常物の結びつきが無理なく感じられる
3. **非明示性**: 「これを見たら思い出して」と言わない
4. **象徴的位置**: 記事内で決定的な場面で登場する

## 失敗条件

- **低頻度物**: 珍しい物を選ぶと想起機会が少ない
- **強制感**: 「思い出して」と明示すると説教臭くなる
- **接続の無理**: 核心と無関係な物を無理に使うと不自然
- **埋没**: 記事内での登場が印象に残らない

## 原型：神社のお札

神社の厄払いにおけるお札は、想起アンカーの原型である。

1. **概念設定** — 厄年という「問題」を定義
2. **解決導線** — 厄払いという「行動」を提供
3. **想起装置** — お札を「頭より高い位置に」→ 日常で視界に入る
4. **再訪トリガー** — お焚き上げで1年後の来訪を予約
5. **内発化** — 「強制」ではなく「自分で選んだ」と感じさせる

神社は「これは想起装置です」とは言わない。「頭より高い位置に置いてください」と言うだけ。

## 適用事例

| 日常物 | 核心との接続 | 記事/コンテンツ |
|--------|-------------|----------------|
| チーズサンドイッチ | 期待の崩壊、愛から憎しみへの反転 | Fyre Festival記事 |
| 給食 | 問いを渡す機会 | Day 45「親の30分」 |
| ゲーム/携帯 | 子どもが考え始める瞬間 | Day 45「親の30分」 |
| 蛇口/シャワー | 変化を止められない | ヘラクレイトス×コダック |
| 信号機 | 停止と進行の判断 | （未使用） |
| 鏡 | 自己認識、傍観者性 | （未使用） |

## 日常物の選定基準

| 基準 | 説明 |
|------|------|
| 毎日接触 | 週1回ではなく、毎日視界に入るもの |
| 普遍性 | 読者の大多数が持っている/見るもの |
| 意味の余白 | 既に強い意味が固定されていない物 |
| 核心との共鳴 | 記事の構造と自然に結びつく |

## 関連パターン

- **framing**: 想起アンカーは日常物をフレーミングする行為
- **substitution**: 日常物の意味が記事の核心で置換される
- **persistence**: 想起が繰り返されることで記憶が持続

## 設計メモ

西野亮廣『北極星』より引用される「仕組み集客」の概念を、記事構成に応用したもの。

> 仕組み集客とは、一度きりの注目を奪う技術ではない。
> 人々の生活や行事の中に、自分たちの居場所を静かに用意する行為だ。
> 新しい流れを作る必要はない。
> すでに流れている水を読めばいい。

記事における想起アンカーは、読者の「生活動線」に記事の存在を静かに埋め込む技術である。
