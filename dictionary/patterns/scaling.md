# scaling（スケーリング）

システム・組織・技術が規模を拡大し、新たな性質や課題が創発する構造。

## 定義

量的拡大が質的変化をもたらす過程。規模が一定の閾値を超えると、それまでとは異なる法則・制約・可能性が出現する。

## 構造

```
initial_scale
  → growth_pressure
    → resource_multiplication
      → threshold_crossing
        → emergent_properties (new constraints/opportunities)
```

## 発動条件

1. 成長への需要または圧力
2. スケール可能なアーキテクチャ
3. 資源（資本・人材・インフラ）の利用可能性
4. 規模の経済の存在

## 失敗条件

- **複雑性爆発**: 管理不能な相互依存
- **スケール不経済**: 大規模化による非効率
- **ボトルネック**: 拡張不能な要素の存在
- **文化崩壊**: 組織規模拡大による価値観の希薄化

## 適用事例

| Layer | Entities |
|-------|----------|
| technology | cloud-computing, data-center |

## 関連パターン

- **network-effects**: スケールがネットワーク効果を強化
- **institutionalization**: 規模拡大に伴う制度化
- **fragmentation**: 過剰スケールによる分裂
