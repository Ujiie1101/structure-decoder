# Self-Tipping（自分転倒）

## 定義

外部からの転倒を待たず、**自分でタイミングと規模を選んで権威を一回毀損する戦略**。転倒のエネルギーを内部で消化することで、外部からの破滅的転倒（authority-inversion）を予防接種する。

## 公式

```
鎖①〜③で痛みが顕在化し始める
   ↓
分岐点：転倒は不可避と理解する／しない
   ↓
理解する場合 → 自分転倒（戦略B）
理解しない場合 → 隠す（戦略A）
   ↓ 戦略A の場合
契約の更新（次の事故も隠さねばならない）
   ↓
外部から非線形に転倒される
```

## メカニズム

### 1. 転倒の不可避性の前提

自分転倒は、「事故は起きうる」「権威の毀損は時間の問題」という前提を持つ組織にしか発動しない。「事故を起こさない」「絶対バレない」という前提を持つ組織は、戦略Aを選ばざるを得ない。

### 2. タイミングと規模のコントロール

| 転倒の種類 | タイミング | 規模 | 修復可能性 |
|---|---|---|---|
| 外部から転倒される | 不可 | 不可 | 致命的になりやすい |
| 自分転倒する | 可 | 可 | 制御された傷で済む |

外部からの転倒は累積した不信の総和を一気に放出する。自分転倒は事故単位で放出するため、振幅が小さい。

### 3. 二層信頼の同時形成

自分転倒は、その事故についての信頼を一回毀損するが、同時に**「この組織は事故を隠さない」という別レイヤーの信頼**を形成する。後者は前者の損失を上回る速度で蓄積する性質がある。

### 4. 隠蔽契約の回避

戦略Aを選ぶと、「隠す自分」を演じる契約が成立する。次の事故も隠さねばならない。契約は更新され続け、降りられなくなる。**自分転倒は、この契約への署名を拒否する動作**である。

## 発動条件

- 信頼の非対称性（線形蓄積／非線形崩壊）を構造として理解している
- 事故が「想定範囲内」に収まると判断できる
- 公開のチャネル（ポストモーテム、開示文化、コミュニケーション線）が事前に存在する
- 経営判断として「短期的恥」を受け入れる耐性がある

## 失敗条件

- 想定を超える事故を開示する → 信頼が逆方向に動く
- 「想定範囲」の線引きそのものが self-undermining-success の対象になる（成功すると線引きの精度が落ちる）
- 開示の頻度が高すぎて陳腐化する
- 開示の文章が形式化（誰も読まないテンプレート化）

## 判別法

1. 事故が起きた後、**ユーザーが大規模に騒ぐ前に**自発開示しているか
2. 開示文に「原因」「再発防止策」「責任の所在」が明示されているか
3. 開示後の信頼曲線が、開示前と比較して**長期的に上向く**か
4. 経営層が「開示しない選択肢」を最初から持っていないか

## 事例

| Layer | Entities |
|-------|----------|
| AI industry | Anthropic（2026-04 Claude Codeポストモーテム自発公開） |
| tech | GitHub・AWS・Stripe等のポストモーテム文化 |
| medical | M&Mカンファレンス（医療事故症例検討会） |
| aviation | 航空事故調査報告書の公開原則（NTSB） |

## 対比事例（自分転倒に失敗した組織）

| 組織 | 失敗内容 |
|---|---|
| コダック | 自社発明のデジタル写真を③で隠した結果、37年後に破産 |
| リーマン・ブラザーズ | サブプライムリスクを③で先送り、④の窓を逃して158年の歴史終了 |
| Theranos | 技術不在を隠し続けた結果、外部告発で90億ドル消滅 |
| LTCM | モデルの想定外事象を③で開示せず、46億ドル連鎖崩壊 |

## 関連パターン

- **authority-inversion**: 自分転倒が予防する対象
- **authority-inversion-absence**: 自分転倒も外部転倒も起きない静的老化状態
- **trust**: 信頼の非対称性が自分転倒の数学的根拠
- **self-undermining-success**: 自分転倒の「想定範囲」線引きが対象になる再帰構造
- **externalization-cascade**: 鎖②③で発生する痛みが自分転倒の発動契機

## 核心洞察

> **隠す者は、契約に縛られる。自分転倒は、その契約への署名を拒否する動作である。**
> **誠実さではない。信頼の崩壊曲線の数学を理解している組織だけが、自分転倒できる。**
