# shortcut（近道）

正規の手順を省略し、より短い経路で目的を達成しようとする構造。

## 定義

標準的なプロセス・手続き・学習過程を迂回し、より直接的・効率的な方法で結果を得ようとする行動パターン。短期的効率と長期的リスクのトレードオフを内包する。

## 構造

```
desired_outcome
  → standard_path (time/effort intensive)
    → bypass_discovery
      → accelerated_result
        → hidden_cost (quality, sustainability, learning)
```

## 発動条件

1. 時間・資源の制約
2. 迂回路の発見
3. 短期的メリットの認識
4. リスク評価の軽視

## 失敗条件

- **発覚**: 迂回が露見しペナルティを受ける
- **品質劣化**: 省略したプロセスの価値が顕在化
- **学習欠如**: 必要なスキル・知識が蓄積されない
- **依存**: 近道が常態化し正規能力が失われる

## 適用事例

| Layer | Entities |
|-------|----------|
| technology | technical-debt, copy-paste-programming |
| economics | ponzi-scheme, pump-and-dump |
| philosophy | sophistry, pseudo-science |
| art | plagiarism, ai-generated-art-controversy |

## 関連パターン

- **fragility**: 近道がシステムを脆弱にする
- **cascade**: 近道の失敗が連鎖
- **feedback**: 近道への依存が強化される

## メカニズムID

**[E-01] Invisible Hand Coordination** — 個々の近道選択から集合的パターンが創発
