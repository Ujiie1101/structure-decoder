# Threshold Compression Pattern（閾値圧縮）

## 定義
外部技術の指数関数的進化が、既存構造の崩壊閾値を継続的に引き下げる現象。ある時点では「5年後に訪れる危機」と見えたものが、閾値圧縮によって「半年後の危機」に変換される。

## 公式
```
既存構造の崩壊閾値 T(t)
  ↓
外部技術の指数関数的進化
  ↓
代替手段のコスト・品質が急速に向上
  ↓
既存構造の相対的価値が急速に低下
  ↓
閾値 T(t+1) < T(t)（閾値そのものが低下）
  ↓
「予想より早く」崩壊が訪れる
```

## メカニズム

### 1. 時間軸の非対称性
既存企業の時間感覚: 四半期・年次計画（**線形**）  
技術進化の時間感覚: ムーアの法則・スケーリング則（**指数関数**）

両者は別の時計を持つ。既存企業の「3年計画」が完了する頃には、技術側は2世代進んでいる。

### 2. 閾値の可動性
従来「閾値」は固定された物理定数のように論じられてきた。しかし現代では、**閾値そのものが技術進化で動く**。

- 2020年 GPT-3登場: 自然言語処理業務の閾値が下がる
- 2023年 GPT-4: 事務作業全般の閾値が下がる
- 2025〜 エージェント時代: 自律業務執行の閾値が下がる

### 3. 観察者の認知ラグ
人間の危機認知は**現時点の閾値**に基づいている。閾値が動いていることを見落とす。結果、「来年は大丈夫」と判断したとき、既に閾値側が移動している。

## 成功条件（記事での提示として）
- 技術進化のタイムスタンプ（いつ何が起こったか）
- 閾値が動いた**数値的根拠**（コスト半減、処理速度10倍等）
- 既存企業の時間軸と技術時間軸の**並列図示**

## 失敗パターン
- 「AI時代が来る」という曖昧な主張 → 閾値圧縮の具体性が失われる
- 技術決定論に偏る → 企業の主体的選択を無視する
- 全産業に一律適用 → 業種差を見落とす

## 事例

### Day 54「時間差崩壊」
- 2022年 ChatGPT公開 → 2024年 事務アウトソーシング閾値が下がる
- 2010年代 クラウド成熟 → オンプレミス型基幹系の閾値低下
- 生成AI登場で「新卒の価値」閾値が圧縮中（2024〜）

### 他分野への応用
- カメラ業界: スマホカメラ進化で「一眼の閾値」が圧縮
- 翻訳業界: DeepL・LLM進化で「翻訳者の価値閾値」圧縮
- 音楽業界: 配信進化で「CDショップの閾値」圧縮

## 関連概念
- **基盤**: threshold（閾値）
- **帰結**: delayed-exponential-collapse（時間差崩壊）
- **要因**: acceleration（加速）、技術進化のS字カーブ連鎖
- **現象の反転**: 新規参入者の参入閾値も下がる（両刃）

## 配置場所（記事での使い方）
- 時間差崩壊の**加速因**として提示
- 「昨日まで5年後の危機が、今日は2年後」のような**時間表記の反復**で印象付け
- 記事後半、読者に危機感を植え付ける段階で使う

## 判別法（業界がこの構造の影響下にあるか）
1. 外部技術（AI、ロボット、バイオ等）の指数関数的進化が進行中か？
2. その技術が、自業界の主要業務を代替可能か？
3. 代替技術のコストが過去2年で半減以上しているか？

3つYesなら閾値圧縮下にある。

## 注意点
- **対称性**: 閾値圧縮は「崩壊」だけでなく「新規参入」も容易にする。破壊者と被破壊者は同じ圧縮下にいる
- **過剰適用禁止**: 全ての業界が指数関数下にあるわけではない（インフラ・規制産業は圧縮速度が遅い）
- **自業界の閾値推定**: 過去データからの外挿ではなく、代替技術側のS字カーブから推定する
