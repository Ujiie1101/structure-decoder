# Title-Number Recovery Pattern（タイトル数字回収パターン）

## 定義
タイトルに入れた数字やキーフレーズを、本文の中盤〜後半で明示的に「回収」することで、記事全体の一貫性と命題の強度を高める手法。

## 公式
```
タイトル: 「[キーフレーズ+数字]」
↓
本文中盤: 「[回収文]＋[タイトルの数字]」
↓
読者の脳内: 「ああ、これがタイトルの意味か」
```

## メカニズム

### 1. タイトル=謎提示、本文=解明
タイトルだけでは意味不明な数字を置く → 読者は「なぜこの数字？」と疑問を持つ → 本文で回収することで謎が解ける。

### 2. 数字の非対称性
1:99、3億:1枚、5000万:3200億 のような**極端な比率**が機能する。
対称的な数字（50:50）は記憶に残らない。

### 3. 命題の強度化
単に「見せしめは社会に影響する」ではなく、「1人を潰すことで99万人を統治する」という数字付き命題は、引用されやすい。

## 成功条件
- タイトルに具体的な数字がある
- 本文中の**特定の1箇所**で明示的に回収する
- 数字の出典/根拠が示されている（または論理的に導かれる）

## 失敗パターン
- タイトルだけ派手で本文に数字が登場しない
- 複数箇所で散漫に数字を使う → 回収ポイントが曖昧に
- 数字が恣意的（根拠不明）

## 事例

### Day 25「彼女は謝った。99万人は黙った。」
```
タイトル: 「99万人は黙った」
↓
本文中盤: 「見せしめは1人を潰すためではなく、
         **99万人を黙らせるため**に機能しています。」
```

### 他の応用例
- タイトル: 「$90億の幻想」→ 本文: 「Theranosは**90億ドル**の評価から0になった」
- タイトル: 「3億のチーズ」→ 本文: 「**3億インプレッション**が**1枚のチーズサンドイッチ**で崩壊した」

## 適用条件
- 構造的問題を数値化できる
- タイトルに数字を入れても違和感がない
- 記事の核心命題が数字で表現可能

## 回収位置
- **推奨**: 全体の50〜70%地点
- **前すぎ**: 謎提示の効果が薄れる
- **後すぎ**: 読者が忘れる

## 注意点
- 数字は**一つに絞る**（複数の数字を回収すると分散する）
- 回収文は**太字**で視認性を確保する
