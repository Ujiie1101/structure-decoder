# trace（痕跡）

不在の現前、消去された起源の残余が意味を可能にする構造。

## 定義

記号の中に刻まれた「他の記号」の痕跡。現前しているものの中に、不在のものの影が残存する。起源なき起源、根拠なき根拠としての差異の堆積。

## 構造

```
sign_A_appears
  → carries_traces_of_absent_B,C,D...
    → absence_constitutes_presence
      → no_pure_presence_possible
        → origin_always_already_effaced
```

## 痕跡の特性

### 不在の現前

- 現在の記号の中に過去の差異が残存
- 「ある」ものの中に「ない」ものが宿る

### 起源の消去

- 「最初の痕跡」は存在しない
- 痕跡は常に「痕跡の痕跡」
- 原-痕跡（arche-trace）という逆説

### エクリチュール

- 痕跡は「書くこと」の構造そのもの
- 音声中心主義の解体

## 発動条件

1. 記号の物質性（inscription）
2. 時間の経過と差異の蓄積
3. 反復可能性（iterability）
4. 文脈からの切断可能性

## 関連パターン

- **differance**: 痕跡を生成する差延の運動
- **supplement**: 痕跡が補遺として機能
- **erosion**: 痕跡の堆積と消失

## 注意

「痕跡」は単なる残り香ではない。存在そのものを可能にする条件でありながら、決して完全に現前しない逆説的な構造。Freudの「神秘のメモ帳」からの着想。
