# transformation（変革）

システム全体が根本的に異なる状態へと移行する構造。

## 定義

漸進的変化ではなく、システムの基本的な性質・構造・原理が質的に異なるものへと変わる。パラダイムシフト、革命、メタモルフォーシスを含む。

## 構造

```
system_state_A
  → accumulating_tensions
    → threshold_breach
      → phase_transition
        → system_state_B (qualitatively different)
```

## 発動条件

1. 旧システムの限界への到達
2. 代替ビジョンの存在
3. 変革を担う主体
4. 移行を可能にする条件

## 失敗条件（変革阻止）

- **抵抗**: 既得権益による阻止
- **能力不足**: 変革実行力の欠如
- **ビジョン欠如**: 代替像の不明確さ
- **時期尚早**: 条件の未成熟

## 関連パターン

- **substitution**: 変革による旧システム置換
- **collapse**: 変革失敗時の崩壊
- **emergence**: 変革からの新秩序創発
